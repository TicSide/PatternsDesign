# How use design patterns?

Well in the repository, you have diferentes folders with diferents design patterns, so you choose that you want to visualize.
For example, if you want to see the design pattern of Singleton, you can go to the folder of Creational Patterns and there you will find the file of Singleton, where you can see the code and the explanation of how it works.

Also I put how use it in the main file *"Program.cs"*, so you can see how to use it in a real project. For example, if you want to use the Singleton pattern, you can see how to create an instance of the Singleton class and how to call its methods.


# Example
- Folder Creational Patterns
- Folder Singlenton
- Folder Examples - Ex-#
- Diferents files with the code of the design pattern and the explanation of how it works.
- A file .md with a summary of the design pattern and its benefits.
- A UML diagram with the structure of the design pattern and its relationships between classes and objects.
- And a file "Program.cs" with the code of how to use the design pattern in a real project.
